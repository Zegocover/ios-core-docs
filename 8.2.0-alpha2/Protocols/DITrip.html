  • rejectedReason(mergeWithServer:)

    Returns the rejected reason or, in other words, the validity state of this trip. The default value is -2, meaning no conclusion has been taken as to if the trip belongs to the user or not.

    Declaration

    Swift

    func rejectedReason(mergeWithServer: Bool) -> Int

    Parameters

    mergeWithServer

    False to retrieve a value without analysis from Drivit server (e.g. the user may reject the trip directly on the phone), true if you want to merge both results. When merging, any potential user input will always have precedence over the analysis from the Drivit server.
